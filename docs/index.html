Real-time Bilingual Translation

An open-source platform for two-way translation with speech recognition, AI-powered translation, and natural voice synthesis.

KK AI Translator was built by Kalundborg Kommune (Denmark) to help caseworkers communicate with citizens who speak different languages. It's now open source and free to use.

The app runs in your browser. Speak into your microphone, and it transcribes, translates, and speaks the translation aloud - in both directions.

Features

Speech-to-Text

Transcribe spoken audio using Azure Speech Services or Whisper. Supports 450+ languages.

AI Translation

Translate between languages using GPT-4o-mini or other LLMs with natural phrasing.

Text-to-Speech

Hear translations spoken aloud with natural-sounding voices via Azure TTS.

Configurable Models

Choose which AI model handles transcription, translation, and summaries for each language.

Session History

Track conversations with full transcripts and AI-generated summaries in both languages.

Easy Setup

Docker Compose gets you running in minutes. Just add your Azure API keys.

Getting Started

What you'll need

  • Docker Desktop - Download here (free)
  • Azure account with Speech Services and OpenAI - Free tier available
  • About 10 minutes

Download the code

Clone the repository to your computer:

git clone https://github.com/kalundborgkommune/KK-AI-Translator-V1-OS.git
cd KK-AI-Translator-V1-OS

Add your API keys

Copy the example environment file and add your Azure credentials:

cp .env.example .env
# Open .env in any text editor and fill in:
# AZURE_SPEECH_KEY=your_key_here
# AZURE_OPENAI_KEY=your_key_here

See SETUP.md for detailed instructions on getting Azure keys.

Start the app

Run this command and wait for it to finish:

docker compose up -d

First run takes a few minutes to download everything.

Open in browser

Go to http://localhost:8080 and start translating!

First time? Visit Settings and click "Seed Languages" to load all available languages.

Deploy to Azure

Ready to deploy to production? There are two approaches depending on your needs:

Option A: Azure VM

Simple deployment using Docker Compose on a Linux VM. Great for getting started quickly with full control.

Option B: Azure Web App

Managed serverless deployment with Azure Container Registry and Azure MySQL. Auto-scaling and high availability.

Quick Start: VM + Docker Compose

The simplest approach - run everything on a single Azure VM:

# Create VM and SSH in
az vm create --resource-group translator-rg --name translator-vm --image Ubuntu2204 --size Standard_B2s

# On the VM: install Docker and clone repo
curl -fsSL https://get.docker.com | sh
git clone https://github.com/kalundborgkommune/KK-AI-Translator-V1-OS.git
cd KK-AI-Translator-V1-OS && cp .env.example .env

# Configure .env with your Azure keys, then start
docker compose up -d

Quick Start: Azure Web App (Serverless)

For production with managed services:

# Build and push to Azure Container Registry
make build-full
make push-full

# Deploy to Azure Web App
make deploy-full

See SETUP.md for detailed step-by-step instructions including Azure MySQL setup, SSL configuration, and continuous deployment.

Configuration

Language Settings

The Settings page (/settings) lets you configure each language:

  • Enable/disable - Choose which languages appear in the translator
  • Voice selection - Pick from multiple voice options per language
  • AI models - Select which model handles transcription, translation, and summaries

Supported Models

Out of the box, the app supports:

  • Azure Speech - Microsoft's speech-to-text (recommended)
  • Whisper via Promte - Alternative transcription endpoint
  • GPT-4o-mini - Fast, affordable translation

Adding new model integrations is straightforward - see the code in python-be/src/services/.
